# Update: January 27, 2026

## Requirement
Kalpesh requested:
- Use `keypoints3D` (3D coordinates in meters, pelvis as origin)
- Apply camera extrinsics for world-space output
- Output in meters, not pixels

## Changes Made
| Item | Before | After |
|------|--------|-------|
| Data source | `keypoints` (2D pixels) | `keypoints3D` (3D meters) |
| Joint names | `right_wrist` | `rightWrist` |
| Camera extrinsics | Not applied | 4x4 matrix transform applied |
| Output space | Local only | World space (meters) |

## Output Format
```json
{
  "position": [x, y, z],        // World space (meters)
  "quaternion": [w, x, y, z],   // World space
  "position_local": [x, y, z],  // Pelvis-local (meters)
  "wrist_position": [x, y, z]   // World space (meters)
}
```
